2. User conversation context if the follow-up came from discussion.
3. Existing backlog/RFC/task relationships that help explain why this item exists.

## Deferred From (Optional)

When this backlog item is intentionally deferred from active task or RFC work, record the in-progress source context here:
1. Source task document(s), for example `docs/tasks/000040-example.md`.
2. Source RFC document(s) or phase references, for example `docs/rfcs/0006-example.md`.
3. Optional branch/worktree or execution context if it materially helps future recovery.

Leave blank for standalone backlog items.

## Deferral Context (Optional)

1. `Defer Reason`: why the work is postponed now.
2. `Findings`: what current execution learned that should not be lost.
3. `Direction Hint`: what future task/RFC planning should revisit or prefer.

Leave blank for standalone backlog items.
